Краткое объяснение алгоритма:
Цель: Найти максимальное независимое множество (набор вершин, где никакие две не соединены ребром) в графе.
Алгоритм локального поиска:
1.	Инициализация: Начинаем со случайного независимого множества
2.	Улучшение: Последовательно проверяем все вершины графа
3.	Добавление: Если вершина не смежна ни с одной вершиной текущего множества - добавляем её
4.	Повторение: Продолжаем пока можно добавлять новые вершины
Ключевые особенности:
•	Жадная стратегия - всегда добавляем первую подходящую вершину
•	Гарантирует корректность результата (только независимые множества)
•	Прост в реализации, но не гарантирует оптимальность

Временная сложность:

Худшая временная сложность: O(V³)
Обоснование:
•	V - количество вершин (вводится пользователем)
•	Внешний цикл while: O(V) итераций
•	Внутренний цикл по вершинам: O(V)
•	Проверка соседей для каждой вершины: O(V) в худшем случае
•	Итого: O(V) × O(V) × O(V) = O(V³)
Почему это худший случай:
•	В полном графе (клике) где каждая вершина соединена со всеми остальными:
•	Можно добавить только 1 вершину в независимое множество
•	Каждая проверка требует просмотра всех V-1 соседей
•	Алгоритм будет проверять все V вершин на каждой итерации
Контрольный вопрос: 15. Локальный поиск
Что такое локальный оптимум и почему метаэвристики нужны для его преодоления?

Локальный оптимум — это решение, которое является оптимальным (лучшим) в пределах своей небольшой "окрестности", но не обязательно является лучшим решением для всей задачи в целом (глобальным оптимумом). 
Метаэвристики — это общие стратегии высокого уровня, предназначенные для нахождения достаточно хороших решений в сложных задачах оптимизации, где точные методы работают неприемлемо долго.
Они нужны для преодоления локальных оптимумов, потому что обладают механизмами, позволяющими целенаправленно ухудшать текущее решение, чтобы в итоге найти лучшее.
Таким образом, метаэвристики — это мощные инструменты, которые жертвуют гарантией нахождения идеального решения ради возможности найти очень хорошее решение за разумное время в задачах, где классические методы бессильны из-за ловушек локальных оптимумов.
